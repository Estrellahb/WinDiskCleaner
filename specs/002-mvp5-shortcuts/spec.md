# Feature Specification: MVP5 快捷方式检测与修复

**Feature Branch**: `mvp5-shortcuts`  
**Created**: 2026-06-25  
**Status**: Draft  
**Input**: WinDiskCleaner MVP5：快捷方式检测与修复

## User Scenarios & Testing

### Primary User Story

用户打开 Tab5 快捷方式页面后，可以扫描桌面、开始菜单、启动项中的 `.lnk` 快捷方式，识别目标不存在的失效快捷方式，确认后批量删除选中的失效项，并导出 CSV 或 HTML 检测报告。

### Acceptance Scenarios

1. **Given** 快捷方式扫描目录为空，**When** 执行扫描，**Then** 返回空列表且统计为 0。
2. **Given** 存在目标有效的 `.lnk` 文件，**When** 执行扫描，**Then** 该项显示为有效。
3. **Given** 存在目标已不存在的 `.lnk` 文件，**When** 执行扫描，**Then** 该项显示为失效并在 UI 中高亮。
4. **Given** 目录中存在非 `.lnk` 文件，**When** 执行扫描，**Then** 非快捷方式文件被忽略。
5. **Given** 扫描过程中遇到无权限目录，**When** 执行扫描，**Then** 该目录被跳过且扫描不中断。
6. **Given** 用户取消扫描，**When** CancellationToken 被触发，**Then** 扫描及时取消。
7. **Given** 用户勾选若干失效快捷方式，**When** 确认删除，**Then** 只删除 `.lnk` 文件本身，不删除目标程序或文件。
8. **Given** 已有扫描结果，**When** 导出报告，**Then** 可生成 CSV 与 HTML 报告。

## Requirements

### Functional Requirements

- **FR-001**: 系统必须扫描当前用户桌面、公共桌面、当前用户开始菜单、公共开始菜单、启动文件夹中的 `.lnk` 文件。
- **FR-002**: 系统必须递归扫描目标目录中的 `.lnk` 文件。
- **FR-003**: 系统必须解析快捷方式目标路径与参数。
- **FR-004**: 系统必须检查目标文件或目录是否存在，并标记有效/失效状态。
- **FR-005**: 系统必须支持取消扫描。
- **FR-006**: UI 必须展示名称、路径、目标路径、状态、来源目录。
- **FR-007**: UI 必须对失效项进行红色高亮。
- **FR-008**: UI 必须支持搜索过滤，并支持按路径、状态排序。
- **FR-009**: 删除操作必须只处理用户勾选的失效项。
- **FR-010**: 删除前必须展示快捷方式路径与目标路径并请求确认。
- **FR-011**: 删除操作必须只删除 `.lnk` 文件本身，不删除目标路径。
- **FR-012**: 删除失败必须记录原因，不能影响其他项继续处理。
- **FR-013**: 删除后必须刷新列表。
- **FR-014**: 系统必须支持导出 CSV 报告。
- **FR-015**: 系统必须支持导出 HTML 报告。

### Safety Requirements

- **SR-001**: 删除候选只能来自目标不存在的失效 `.lnk` 项。
- **SR-002**: 有效快捷方式不得进入删除候选。
- **SR-003**: 删除执行前必须再次校验文件扩展名为 `.lnk`。
- **SR-004**: 删除执行不得触碰快捷方式指向的目标程序或文件。
- **SR-005**: 用户确认内容必须包含快捷方式路径与目标路径。

### Key Entities

- **ShortcutItem**: 快捷方式扫描项，包含名称、快捷方式路径、目标路径、参数、目标是否存在、来源目录、文件大小、修改时间、是否选中。
- **ShortcutScanResult**: 扫描输出，包含快捷方式列表与统计信息。
- **ShortcutDeleteResult**: 删除输出，包含成功项、失败项与失败原因。
- **ShortcutReport**: 导出报告内容，支持 CSV 与 HTML 格式。

## Out of Scope

- 不做注册表快捷方式清理。
- 不修复或重建快捷方式目标路径。
- 不扫描开始菜单以外的自定义位置。
- 不做安装包。
- 不做自动清理。
